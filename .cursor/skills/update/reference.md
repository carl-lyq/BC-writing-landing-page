# Update — 通用参考

## 主源识别清单（Step 0）

在 README、PRD 文首、交付说明中查找下列信号：

| 信号词 | 通常角色 |
|--------|----------|
| 「唯一来源」「工作稿」「持续更新」 | 主 PRD |
| 「开发冻结」「定稿」「验收以本文为准」 | 开发 PRD（可能需单独同步） |
| 「归档」「勿验收」「Demo 稿」「v0.x 历史」 | 归档，不更新 |
| 「交付页」「UI 主参考」「index.html」 | 主 HTML |
| 「Demo.html」「历史原型」 | 非基准 |
| 「交付包」「交付开发」 | 可能是镜像目录，核对是否与工作区同步 |

**向用户确认模板**（主源不明时）：

> 本项目我识别到多个 PRD/HTML：
> - PRD A: …
> - PRD B: …
> - HTML X: …
> 请确认：**主 PRD** 与 **主 HTML** 各是哪一个？开发冻结稿/交付包是否需要本次一并同步？

## 变更分类速查

| 类型 | 主 PRD | 水滴 | TODO |
|------|--------|------|------|
| 文案定稿 | ✅ | ✅ 缩短 body | — |
| 交互/规则 | ✅ + 验收标准 | ✅ | 若阻塞则 ✅ |
| 待办新增 | ✅ 变更记录 | ✅ TodoDrip | ✅ |
| 待办完成 | ✅ 去「待定」 | 更新/移除 | ✅ |
| 调试工具 | ✅ 生产剔除 | ❌ | — |
| 纯 CSS 微调 | 视可读性 | 可选 | — |

## NoteDrip / TodoDrip 模板

```jsx
<DripAnchor side="end" drip={
  <NoteDrip
    inline
    title="简短标题"
    body={"要点 1\n· 要点 2"}
    prd={["§2.2 功能 N"]}
    docs={["prd", "todo"]}
  />
}>
  {/* 锚定控件 */}
</DripAnchor>
```

```jsx
<DripAnchor side="end" drip={<TodoDrip inline todoId="T-001" />}>
  <button>…</button>
</DripAnchor>
```

多水滴并列：`drip={<><NoteDrip … /><TodoDrip … /></>}`

## PRD §非生产 声明要点

- 蓝色 `NoteDrip` = PRD/逻辑备注
- 橙色 `TodoDrip` = 待办
- `proto-toolbar` / Tweaks = 调试，默认收起
- **生产构建须剔除**上述全部

---

## 交叉校验扩展

- `DOC_LINKS` 中 `prd` / `todo` 路径指向**当前项目主文档**
- 交付包内 HTML 的 `DOC_LINKS` 应相对路径可达主 PRD/TODO
- 冻结 PRD 版本号与变更记录是否与用户预期一致
